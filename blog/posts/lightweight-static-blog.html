2G 机器也够用：轻量博客怎么选

资源有限时，最省的不是「更小的 CMS」，而是根本不跑 CMS。

手里有台 2 核 2G，想加个人博客，常见选项大致这样排：

方案对照

  • 静态 HTML / Hugo / Astro：几乎零内存，CDN 托管即可 —— 本站采用
  • Typecho：PHP 轻量后台，适合必须在服务器上点点写的人
  • WordPress：能跑，但插件一多就挤占 2G
  • Ghost / Halo：Node / JVM 常驻，小机器容易吃紧

本站的取舍

主页已经在 Cloudflare Pages。博客做成同风格静态页后：

  1. 不需要再在 VPS 上开 Nginx + PHP/MySQL
  2. 2G 留给 Uptime、股票分析、其它自托管服务
  3. 视觉继续用 NES + Fusion Pixel，和 HOME / ITEM BAG 一体
什么时候才上 Typecho？——当你明确需要后台编辑、评论审核，并且愿意在机器上留出 PHP 进程时。

小结

轻量化的第一原则：能静态就静态。博客写进仓库，推送即发布，机器呼吸更顺。
