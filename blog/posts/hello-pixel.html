像素主页开张：先存一档

主页是仪表盘，博客是任务日志。两者共用同一套像素皮肤，部署还是 Cloudflare Pages。

这是 Pixel Home 的第一篇存档。

站点本身是纯静态页：NES 面板、Fusion Pixel 字体、WORLD MAP 上的工具链。博客也走同一条路——不跑数据库、不占 2G 机器的常驻内存，写完 HTML 推到仓库即可。

为什么是静态

  • 和现有主页同仓、同风格、同域名路径 /blog/
  • Cloudflare Pages 构建命令可留空，输出目录就是仓库根
  • 那台 2 核 2G 可以留给真正需要进程的服务

怎么继续写

  1. 在 blog/posts/ 复制一篇现有文章改内容
  2. 文件名即 slug，例如 my-note.html
  3. 在 blog/posts.json 加一条元数据（标题、日期、标签、摘要）
  4. 提交推送，Pages 自动发布
提示：列表页会读 posts.json；即使 JS 失败，也有静态回退卡片。
